# 数据库设计初版

## 核心表

- users
- goods
- goods_images
- exchange_requests
- system_messages
- announcements
- penalties
- sensitive_words
- reports
- admin_logs

## 说明

- 用户展示用数字编号 `user_no`，从 `10000000` 开始
- 商品独立图片表，避免多图字段拼接
- 管理操作必须留审计日志

## 当前已落地的第一张表

### users

- `id`：数据库主键
- `user_no`：展示给用户看的数字编号
- `openid`：微信身份唯一标识
- `unionid`：预留字段
- `nickname`：昵称
- `avatar_url`：头像
- `gender`：`UNKNOWN / MALE / FEMALE`
- `contact_type`：`WECHAT / QQ / MOBILE`
- `contact_value`：联系方式内容
- `role`：`USER / ADMIN`
- `status`：`NORMAL / DISABLED / BANNED`
- `is_profile_complete`：是否已完成资料补全
- `created_at` / `updated_at` / `last_login_at`
